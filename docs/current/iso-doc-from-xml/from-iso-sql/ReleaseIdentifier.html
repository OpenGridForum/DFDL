Release identifier

The release identifier uniquely identifies published versions of a standard

It consists of

  • The number of the standard (e.g. 9075, 19075, 39075).
  • An underscore (_).
  • The generation number (e.g. 8).
  • The stage, i.e. docStage, e.g., IWD. Please see StandardParts.dtd for the list of possible values.
  • The per-stage version number (e.g. 3).
  • A hyphen (-).
  • If the document is part of a multipart standard then:

    The two digit part number.
    A hyphen (-).

  • The short name of the standard or part.
  • An underscore (_).
  • The date in ISO 8601 format.

For example:

  • 9075_8IWD18-16-PGQ_2021-10-15
  • 39075_1IWD18-GQL_2021-09
